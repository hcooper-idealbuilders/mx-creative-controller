# 2026-05-13 (closer) — Shipping v0.2

After the Col03 breakthrough and the v0.1 controller, we kept pushing through the same session.

## v0.1 in production: where it broke

Once the keypad started running for real, two issues surfaced:

1. **Hooks weren't actually firing.** The settings.json hook command had backslash-encoded paths (`C:\\Users\\...`). After JSON decoding to a single backslash and passing through the shell, the backslashes were getting stripped — Claude Code was trying to run `C:Usershdcooper...update-status.ps1` (no path separators). Fix: forward slashes in the path. Windows PowerShell happily accepts them.

2. **`send-keys.ps1` had a `$Pid` parameter** which collides with PowerShell's read-only automatic `$PID`. Every keystroke command died at parameter binding with "Cannot overwrite variable Pid because it is read-only or constant." Renamed to `$ProcessId` and presses started landing in the terminal.

These were both 30-second fixes once we noticed them. The lesson: **end-to-end testing immediately** — we wrote both bugs days apart but didn't catch either until we actually pressed a key on the hardware. Type-checking and pipe-testing in isolation missed both.

## Multi-session refactor

Hunter's dream layout was 3 columns × 3 rows — one column per Claude session, FIFO. Row 0 status mark, row 1 primary (Continue / Approve / Resume), row 2 secondary (Focus / Dismiss). Smart Continue: when state is `waiting_input`, it sends `y⏎` instead of `continue⏎` — same button handles permission prompts.

Architecture changes:

- **Hooks now write per-session files** in `~/.claude/mx-sessions/<session_id>.json`. Added a `SessionEnd` hook for the `ended` state (red mark).
- **Sidecar watches a directory** instead of a single file. Maintains a sorted (by `first_seen`) sessions array, broadcasts `{ type: 'sessions', sessions: [...] }` on every change. New `dismiss` command deletes a session's file.
- **Keypad renders 9 keys** as a 3×3 grid. Column N renders `sessions[N]`; empty columns are dark. Press dispatch maps key index → (col, row) → command per session state.

For the **thinking** state, the keypad renders two pulsing dots in place of the Claude mark. Animation cadence: 200ms refresh while any session is thinking, 1000ms otherwise (still reclaims from Logi Options+'s app-focus repaints).

## Packaging

`install.ps1` + `uninstall.ps1` at the repo root:
- **install.ps1** self-elevates via UAC, builds both packages, generates per-task launcher scripts in `launchers/`, registers `mx-sidecar` and `mx-keypad` as Windows Scheduled Tasks (start at user logon, restart on failure, hidden window, log to `logs/`). With `-StartNow` it kills any in-repo dev `node` processes and starts the tasks immediately for clean handover.
- **uninstall.ps1** stops and unregisters both tasks; leaves source/build/logs.

One install bug I should mention because it cost a redo: the original install.ps1 tried to inline the full hidden-PowerShell command into the Scheduled Task's `Arguments` string. The nested quotes around paths-with-spaces (`"C:\Program Files\nodejs\node.exe"`) broke PowerShell's command-line parsing — the task registered fine but exited immediately with no logs. The fix was to write a small per-task launcher script and have the Scheduled Task action just invoke `-File <launcher>` — way less quoting to get wrong.
